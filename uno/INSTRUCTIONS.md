# Uno

We'd like you to implement the game of Uno.

This is a freeform project. Its purpose is two-fold: to get you thinking independently and creatively, and to be a benchmark for you to reflect on later in the course.

Some tips:

- A big part of programming is psychology. Always be searching for ways to make the project more enjoyable for yourself.
- Online guides will likely add a lot of unnecessary complexity to the project. Part of your job as an engineer is to invent simplicity.
- A game should be fun to play.
- Your interpretation of the rules is up to you, as long as you can justify it. Simplicity is a good justification.
- It's always good to write idiomatic Python code. Maybe have a glance at the [Zen of Python](https://peps.python.org/pep-0020/)?

For this project, we'll be doing *peer reviews*, where you'll study, play, and break each other's games. Be ready to present your work, and think about what you'd like to see in others' projects.

Good luck.
